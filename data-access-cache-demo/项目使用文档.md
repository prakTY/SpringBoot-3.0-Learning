# 数据访问与缓存综合项目

## 项目概述

本项目是一个基于Spring Boot 3.0的数据访问与缓存综合演示项目，展示了企业级应用中数据访问层的最佳实践。项目集成了多种数据访问技术和缓存解决方案，为开发者提供了完整的技术参考。

## 技术栈

### 后端框架
- Spring Boot 3.0.0
- Spring Data JPA 3.0
- Spring Data MongoDB
- Spring Data Redis
- MyBatis 3.0.3

### 数据库
- MySQL 8.0 (主数据源和次数据源)
- MongoDB 4.4+ (文档数据库)
- H2 (测试数据库)

### 缓存技术
- Redis (分布式缓存)
- Memcached (内存缓存)

### 开发工具
- Maven 3.6+
- Lombok
- JUnit 5
- HikariCP (连接池)

## API接口说明

### 用户管理API

#### 基础CRUD
- `GET /api/users` - 获取所有用户
- `GET /api/users/{id}` - 根据ID获取用户
- `POST /api/users` - 创建新用户
- `PUT /api/users/{id}` - 更新用户信息
- `DELETE /api/users/{id}` - 删除用户

#### 查询功能
- `GET /api/users/username/{username}` - 根据用户名查询
- `POST /api/users/search` - 动态条件查询

### 产品管理API

#### MongoDB操作
- `GET /api/products` - 获取所有产品
- `GET /api/products/{id}` - 根据ID获取产品
- `POST /api/products` - 创建新产品
- `PUT /api/products/{id}` - 更新产品信息
- `DELETE /api/products/{id}` - 删除产品

#### 高级查询
- `GET /api/products/category/{category}` - 按分类查询
- `GET /api/products/price-range` - 价格范围查询
- `GET /api/products/low-stock` - 低库存产品
- `GET /api/products/aggregate` - 聚合统计

## 快速开始

### 1. 数据库准备

#### MySQL数据库
```sql
-- 创建主数据库
CREATE DATABASE primary_db;

-- 创建次数据库
CREATE DATABASE secondary_db;

-- 创建用户表
CREATE TABLE users (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL,
    age INT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

#### MongoDB数据库
```javascript
// MongoDB会自动创建集合，无需预先创建
```

#### Redis配置
```bash
# 启动Redis服务

# 验证连接
redis-cli ping
```

### 2. 配置修改

修改`application.properties`文件中的数据库连接信息：

```properties
# 主数据源配置
spring.datasource.primary.url=jdbc:mysql://localhost:3306/primary_db
spring.datasource.primary.username=your_username
spring.datasource.primary.password=your_password

# 次数据源配置
spring.datasource.secondary.url=jdbc:mysql://localhost:3306/secondary_db
spring.datasource.secondary.username=your_username
spring.datasource.secondary.password=your_password

# MongoDB配置
spring.data.mongodb.host=localhost
spring.data.mongodb.port=27017
spring.data.mongodb.database=mongo_db

# Redis配置
spring.redis.host=localhost
spring.redis.port=6379
```

## 性能优化建议

### 数据库优化
1. **索引优化**
   - 为常用查询字段添加索引
   - 使用复合索引优化多条件查询
   - 定期分析查询性能

2. **查询优化**
   - 使用分页查询避免大量数据加载
   - 避免N+1查询问题
   - 使用批量操作减少数据库访问

3. **连接池配置**
   - 合理设置连接池大小
   - 配置连接超时时间
   - 监控连接池使用情况

### 缓存优化

1. **缓存策略**
   - 合理设置缓存过期时间
   - 使用合适的缓存淘汰策略
   - 实现缓存预热机制

2. **缓存一致性**
   - 更新数据时同步更新缓存
   - 删除数据时清除相关缓存
   - 考虑缓存穿透和雪崩问题

3. **缓存监控**
   - 监控缓存命中率
   - 统计缓存使用情况
   - 及时调整缓存配置

### 代码优化

1. **批量操作**
   - 使用批量插入减少数据库访问
   - 批量更新提高效率
   - 批量删除优化性能

2. **异步处理**
   - 使用@Async异步执行耗时操作
   - 合理使用线程池
   - 避免阻塞主线程

3. **资源管理**
   - 及时关闭数据库连接
   - 合理使用事务
   - 避免内存泄漏

## 常见问题解决

### 1. 数据库连接问题

**问题**: 无法连接到MySQL数据库

**解决方案**:
- 检查MySQL服务是否启动
- 验证连接字符串格式
- 确认用户名和密码正确
- 检查防火墙设置

### 2. JPA查询性能问题

**问题**: 查询执行缓慢

**解决方案**:
- 添加适当的数据库索引
- 使用JOIN FETCH避免N+1问题
- 考虑使用原生SQL查询
- 启用SQL日志分析查询

### 3. 缓存一致性问题

**问题**: 缓存与数据库数据不一致

**解决方案**:
- 更新操作时清除缓存
- 使用@CachePut更新缓存
- 设置合理的缓存过期时间
- 实现缓存刷新机制

### 4. MongoDB连接问题

**问题**: 无法连接到MongoDB

**解决方案**:
- 检查MongoDB服务状态
- 验证连接配置
- 确认网络连通性
- 检查MongoDB版本兼容性
